# Final_project_GPT
##GPT 구현 요약 보고서

이번 과제에서는 notebook 1부터 6까지 단계적으로 확장해 온 문자 단위 language model 을 하나의 작은 GPT 구조로 통합하고, "Attention Is All You Need" 논문의 일부를 참고하여 구현했다. 학습 데이터는 tiny Shakespeare 대신 셜록 홈즈(The Adventures + The Memoirs)를 사용하였다.

notebook 1 의 bigram 모델은 Dataset → DataLoader → Model → Loss → Train loop → Sampling 이라는 재사용 가능한 학습 구조를 정의했고, 이번 코드의 train_one_epoch 과 sample_gpt 는 이 골격을 거의 그대로 가져온 것이다. notebook 2 는 one-hot 대신 embedding 으로 문자를 표현하는 방식을, notebook 4 는 target 을 다음 문자 1 개가 아니라 시퀀스 y=[t2, …, t(T+1)] 로 두고 출력 shape 을 (B, T, V) 로 만드는 인터페이스를 도입했다. 

self-attention 의 단일 헤드 형태는 notebook 5 에서 이미 완성되었다. 입력 x 의 shape 은 (B, T, C) = (64, 64, 128) 이며, Head 는 이를 key, query, value 세 개의 선형변환으로 각각 (B, T, head_size) = (64, 64, 32) 로 projection한다. head_size 는 emb_dim 을 head 개수로 나눈 값(128 / 4 = 32)이다. 이때 attention 가중치는 wei = q @ kᵀ 로 계산되어 (B, T, T) 형태가 되고, 여기에 1/√d_k 스케일을 곱한다. 이 부분은 논문 3.2.1 의 정의 Attention(Q, K, V) = softmax(QKᵀ / √d_k)·V 를 그대로 옮긴 것이며, 코드의 k.size(-1) ** -0.5 가 정확히 1/√d_k 에 해당한다(d_k = 32). 논문은 d_k 가 커지면 내적 값이 커져 softmax 가 기울기가 거의 0 인 영역으로 밀려나기 때문에 이를 상쇄하려고 √d_k 로 나눈다고 설명하는데, 코드의 스케일링은 바로 이 한 줄의 구현이다. 이어서 하삼각 행렬 tril 로 미래 위치를 -inf 로 막은 뒤 softmax 를 마지막 축(키 위치)에서 정규화하므로, 각 위치는 자신보다 앞선 위치들에만 가중치를 분배한다. 이는 notebook 5 의 masked self-attention 이자, 논문 3.1 이 위치 i 의 예측은 i 이전 위치에만 의존해야 한다고 명시한 조건과 일치한다.

notebook 6 은 이 단일 헤드 위에 multi-head attention, feedforward, residual connection, layer normalization, block stacking 을 얹어 GPT 본체를 완성했고, 이번 코드의 MultiHeadAttention 부터 GPT 클래스까지가 이에 대응한다. MultiHeadAttention 은 (B, T, 32) 를 내놓는 head 4 개를 concat 해 (B, T, 128) 로 복원한 뒤 proj 로 한 번 섞는다. 4 × 32 = 128 = C 이므로 여러 head 로 쪼갰던 표현이 다시 임베딩 차원으로 합쳐진다. FeedForward 는 128 → 512 → 128 의 위치별 MLP 로, 논문 3.1 이 말한 두 번째 sub-layer 인 point-wise fully connected feed-forward network 에 해당한다. Block 은 이 두 sub-layer 를 residual connection 과 layer normalization 으로 감싸며, notebook 6 과 동일하게 정규화를 sub-layer 앞에 적용하는 pre-LN 구조, 즉 x = x + sa(ln1(x)) 와 x = x + ffwd(ln2(x)) 를 그대로 따른다. residual connection 과 layer normalization 을 함께 사용하고 잔차 연결이 성립하도록 모든 sub-layer 와 embedding 의 출력 차원을 d_model 로 통일한다는 점은 논문 3.1 의 원칙과도 부합한다. 이번 코드의 모든 연산이 (B, T, 128) 을 입력받아 (B, T, 128) 을 반환하므로 잔차 덧셈이 차원 충돌 없이 성립한다.

전체 모델은 token embedding 으로 (B, T) 를 (B, T, 128) 로 올리고, notebook 6 과 동일하게 학습되는 position embedding(nn.Embedding)으로 위치 정보를 (T, 128) 형태로 만들어 더한 뒤, Block 4 개를 직렬로 통과시키고, 마지막 LayerNorm 을 거쳐 lm_head 로 (B, T, V) 의 logit 을 출력한다. 즉 emb_dim 인 C=128 차원은 입력 embedding 부터 마지막 정규화까지 모델 전체를 관통하며 보존되고, 오직 lm_head 에서만 vocab_size 차원으로 펼쳐진다. 이 출력 형태는 notebook 6 에서 확인한 logits.shape = (64, 64, vocab_size) 와 정확히 같다. 손실은 모든 위치의 다음 문자 예측을 한꺼번에 평가하고, 학습은 notebook 6 과 동일하게 AdamW(lr=3e-4)와 epoch 당 max_steps=300 으로 진행한다.

전반적으로 이번 과제의 핵심은 notebook 1~5 가 만든 데이터·손실·학습·샘플링의 골격 위에 notebook 6 의 GPT 본체를 그대로 얹고, 그중 attention 의 수식적 정의(scaled dot-product attention 과 √d_k 스케일링)와 residual·layer normalization 의 일반 원칙만을 논문으로 보강하는 것이다. 차원의 관점에서 보면 C=128 이 모델 전체를 일관되게 흐른다는 한 문장으로 구조 전체를 꿸 수 있으며, 이는 notebook 6 이 모든 sub-layer 와 embedding 의 출력을 같은 emb_dim 으로 맞춰 잔차 연결이 성립하도록 구성한 결과이다.
